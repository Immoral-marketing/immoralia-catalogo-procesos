# Deep Knowledge: Facturación automática de clientes (A1)

## Contexto
Este proceso está diseñado para la **facturación general de la empresa a sus clientes finales**. Se trata de automatizar el flujo de ingresos (Revenue) y ventas por los servicios o productos prestados. **No tiene relación con pagos a colaboradores o freelance.**

## Lógica Técnica
Este proceso utiliza n8n para conectar una base de datos de servicios (generalmente Google Sheets) con un software de facturación como tu ERP/software de gestión.
1.  **Trigger**: Cambio de estado en una fila de Sheets (ej. "Listo para facturar") o una ejecución periódica mensual.
2.  **Procesamiento**: Un nodo de n8n calcula los importes finales basados en parámetros fijos y variables (inversión, porcentaje de fee).
3.  **Acción**: Crea automáticamente una factura en estado "Borrador" en tu ERP/software de gestión para revisión humana.
4.  **Notificación**: Envía un mensaje por tu canal de comunicación preferido o tu canal de comunicación preferido al responsable de facturación.

## Beneficios y ROI
-   **Ahorro de tiempo**: Reduce el tiempo de facturación mensual de días a minutos.
-   **Precisión**: Elimina errores de cálculo manual en fees variables.
-   **Escalabilidad**: Factura a 100 clientes con el mismo esfuerzo que a 1.

## Casos de Uso
-   **Agencias de Marketing**: Cálculo de fees basado en la inversión publicitaria gestionada.
-   **Consultoras**: Facturación por hitos o entregables variables.

## Requisitos Previos
-   Cuenta en tu ERP/software de gestión o software compatible.
-   Hoja de servicios estructurada (Template proporcionado por Immoralia).
-   API Key de las herramientas.

## Preguntas Frecuentes
-   **¿Se envían solas al cliente?** No por defecto. Se crean en borrador para asegurar una última revisión humana.
-   **¿Qué pasa si hay un error en los datos?** El sistema detecta inconsistencias y envía una alerta en lugar de crear la factura.
